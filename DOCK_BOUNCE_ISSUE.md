# Dock bounce issue (macOS)

## Симптом
Периодически в доке подпрыгивает иконка второго экземпляра приложения. Окно не открывается.

## Что уже попробовали
- Добавили проверку доступности CDP (`/json/version`) перед запуском browser-use — не помогло.

## Вероятные причины (в порядке приоритета)

### 1. CEF helper-процессы (наиболее вероятно)
CEF на macOS спавнит дочерние процессы: `Helper`, `Helper (GPU)`, `Helper (Renderer)`.
В собранном `.app`-бандле они живут внутри него с `LSBackgroundOnly=true` в Info.plist.
В dev-режиме (`pnpm tauri dev`) полноценного бандла нет, хелперы могут появляться в доке.

**Как проверить:** прыгает ли иконка при открытии трей-браузера (ещё до запуска агента)?
Если да — однозначно CEF helpers.

**Куда копать:** настройки хелперов в CEF-форке Tauri, `tauri.conf.json` → `bundle`, возможно нужен отдельный `Helper.app` с правильным Info.plist.

### 2. Playwright запускает собственный Chromium (запасной вариант)
Если CDP-соединение browser-use не устанавливается, он может сфолбэчиться на локальный Chromium.
На macOS это даёт иконку в доке.

**Как проверить:** прыгает ли иконка только после нажатия Run Agent?

**Куда копать:** логи browser-use, добавить `is_local=False` в `BrowserSession`, или явно передавать уже живое соединение.

## Следующие шаги
1. Определить момент: до или после запуска агента?
2. Если CEF — изучить как в CEF-форке tauri настраиваются хелперы для dev-режима.
3. Если Playwright — добавить `PLAYWRIGHT_BROWSERS_PATH` и убедиться что он не ищет браузеры.
4. Как последний резерв — single-instance plugin, но он симптом лечит, не причину.
